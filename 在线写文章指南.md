# 📝 在线写文章指南

## 🌟 推荐方法

### 方法1: GitHub 网页编辑器(最简单)

#### 步骤:

1. **访问仓库**  
   https://github.com/konghenfuxue-cpu/konghenfuxue-cpu.github.io

2. **进入 `_posts` 目录**

3. **创建新文件**  
   点击 **Add file** → **Create new file**

4. **命名文件**  
   格式: `YYYY-MM-DD-标题.md`  
   示例: `2025-11-05-my-article.md`

5. **使用模板**  
   复制 `_drafts/article-template.md` 的内容

6. **编写内容**

7. **预览**  
   点击 **Preview** 标签查看效果

8. **提交**  
   填写提交信息,点击 **Commit changes**

✅ **完成!** 1-3分钟后文章自动发布

1. 在 GitHub 上进入 `images` 目录

## 📝 Front Matter 说明

每篇文章开头的配置:

```yaml
---
title: '文章标题'           # 必需
date: 2025-11-05            # 必需,格式 YYYY-MM-DD
permalink: /posts/2025/11/url/  # 可选,自定义URL
tags:                       # 可选,文章标签
  - 标签1
  - 标签2
categories:                 # 可选,文章分类
  - 分类名
excerpt: "摘要"             # 可选,文章摘要
---
```

### 常用标签:

**技术类:**
- STM32, Arduino, 嵌入式
- Python, C/C++, 编程
- 学习笔记, 项目总结

**散记类:**
- 散记, 随笔, 思考
- 生活感悟, 读书笔记

### 常用分类:

## 💡 实用技巧

### 1. 使用草稿

将未完成的文章放在 `_drafts` 目录,不会被发布。

完成后移动到 `_posts` 目录。

### 2. 设置发布时间

```yaml
date: 2025-11-10 15:30:00
```

文章会在指定时间后才显示。

### 3. 固定链接

```yaml
permalink: /posts/my-article/
```

自定义文章URL,更美观。

### 4. 添加目录

在文章中添加:

```markdown
* TOC
{:toc}
```

自动生成目录。

### 5. 隐藏文章

```yaml
published: false
```

文章不会被发布,但保留在仓库中。

## 🆘 常见问题

### Q: 文章不显示?

**A:** 检查:
- 文件名格式是否正确(YYYY-MM-DD-title.md)
- Front Matter 是否正确
- 日期是否是未来时间
- Actions 是否成功运行

### Q: 图片不显示?

**A:** 检查:
- 图片路径是否正确
- 图片是否上传到仓库
- 使用相对路径: `/images/pic.png`

### Q: 修改后不更新?

**A:** 
- 等待3-5分钟
- 清除浏览器缓存
- 检查 Actions 部署状态
